## **1. 게임 컨셉 (High Concept)**

* **High Concept**
  * **대시(Dash)** 와 **밟기(Stomp)** 를 통해 적을 밀어내며 전진하는 2D 러너 액션 게임입니다.
* **기존 유명 게임과의 차이점**
  * **쿠키런과의 차이점**: 쿠키런은 점프와 슬라이딩 중심의 단순 회피형 러너이지만, 본 게임은 적을 직접 타격(대시)해 날려 보내거나 머리를 밟아 튕겨 오르는 능동적이고 하드코어한 액션을 유도합니다.
  * **소닉과의 차이점**: 스프링 도약이나 추락 발판 기믹을 차용했으나, 터치 즉시 발동하는 쿨타임 없는 대시 체이닝과 배터리 수집형 피버 모드로 훨씬 빠른 템포의 속도감과 공중 연계 액션을 지향합니다.
* **핵심 메카닉**
  * 화면 터치 시 전방으로 대시하여 적을 강하게 밀쳐냅니다. (대시 연속 연계 가능)
  * 낙하 중 적의 머리를 밟으면 반동 점프로 다시 떠오릅니다.
  * 패턴 문자열 기반의 발판 생성으로 무한히 이어지는 지형을 구성합니다.
  * 대시 중에는 리본 형태의 **Dash Trail** 이펙트가 생겨 속도감을 보강합니다.
  * [3차 추가] 배터리 5개 수집 시 무적 질주와 낙사 보상을 제공하는 **피버 모드(Fever Mode)**가 발동됩니다.
  * [3차 추가] 삼각함수 왕복 운동을 하는 **움직이는 발판**, 밟으면 0.8초 후 추락하는 **무너지는 발판**, 밟으면 높이 도약하는 **트램펄린 발판**이 추가되었습니다.

## **3. 예상 게임 실행 흐름**

![예상 플레이 화면](docs/images/gameplay.png)

* **게임 시작**: `MainActivity`가 실행되어 `TitleScene`을 로드하고, 터치 시 `MainScene`으로 진입하여 시작합니다.
* **인게임**: 캐릭터는 자동으로 점프하며, 화면을 터치하면 오른쪽으로 연속 대시합니다.
* **월드 진행**: 플레이어가 기준 위치를 넘어가려 하면 플레이어는 화면 안에 유지되고, 배경과 발판이 왼쪽으로 이동합니다.
* **전투 및 기믹 액션**
  * 대시 중 적과 충돌하면 적이 회전하며 화면 밖으로 날아갑니다. 머리에 가시가 돋친 SpikyEnemy는 대시로만 격파 가능합니다.
  * 적의 머리를 밟으면 플레이어가 다시 튀어 오릅니다. 트램펄린 발판을 밟으면 더 높이 탄성 점프합니다.
  * 밟은 무너지는 발판은 0.8초 후 흔들리며 아래로 추락합니다.
  * 일반 상태에서 적이나 가시 장애물의 옆면과 부딪히거나 화면 아래로 추락하면 게임오버가 됩니다.
* **결과 화면**: 게임오버 시 점수를 표시하고, 터치하면 즉시 재시작합니다.

## **6. Activity 구성**

```text
MainActivity
└─ BaseGameActivity 상속
   ├─ 디버그 그리드, FPS, 디버그 정보 차단 (배포 버전 false 강제)
   ├─ GameMetrics 크기 설정: 900 x 1600
   ├─ SoundEffects (SoundPool) 및 HighScoreManager 초기화
   └─ createRootScene()에서 TitleScene 생성
```

`MainActivity`는 Android 앱의 진입점입니다. 수업에서 만든 `BaseGameActivity`를 상속하고, `createRootScene()`에서 시작 화면인 `TitleScene`을 만들어 게임을 실행합니다.

## **7. Scene 구성 및 전환 관계**

```text
앱 실행
└─ MainActivity.createRootScene()
   └─ TitleScene (시작 화면)
      └─ 터치 시 ──> MainScene
          ├─ RUNNING (게임 진행)
          │   ├─ Pause 버튼 터치 ──> PauseScene (일시정지 오버레이)
          │   │                         ├─ Resume ──> MainScene 재개
          │   │                         ├─ Restart ──> MainScene 재작동
          │   │                         └─ Exit ──> TitleScene 복귀
          │   └─ 사망 / 추락 ──> GAME_OVER
          └─ GAME_OVER (결과 화면)
              └─ 화면 터치 시 SceneStack.change(MainScene(gctx))로 재시작
```

`TitleScene`에서 터치하면 `MainScene`이 실행됩니다. 게임 진행 중 일시정지 버튼을 누르면 인게임 화면 위에 `PauseScene`이 반투명하게 오버레이 렌더링되며, 게임오버 상태에서 화면을 터치하면 `SceneStack.change()`를 통해 새로운 게임이 시작됩니다.

## **9. 구현하면서 어려웠던 점**

### **1. 플레이어 복귀와 월드 스크롤 보간**
가장 어려웠던 부분은 **플레이어 복귀와 월드 스크롤 보간**입니다. 플레이어가 대시로 기준 위치를 넘었을 때는 플레이어를 화면 안에 유지하면서, 초과 이동량만큼 배경과 발판을 왼쪽으로 이동시켜야 했습니다.
처음에는 기준 위치를 넘는 순간 배경과 발판을 바로 이동시켰기 때문에 화면 전환이 끊겨 보이거나, 대시 후 플레이어가 원래 위치로 돌아오는 동작이 어색했습니다. 그래서 플레이어의 화면 위치, 실제 이동량, 배경과 발판의 스크롤 양을 함께 계산하고, 복귀와 스크롤에 보간을 적용해 더 부드럽게 보이도록 조정했습니다.

### **2. [3차 추가] 움직이는 플랫폼 탑승 오브젝트 동기화**
움직이는 발판 위에 적이나 가시, 아이템이 고정된 좌표만 가질 경우, 발판만 혼자 왕복 운동을 하고 위에 얹어진 물체들은 허공에 뜨거나 발판 속으로 파묻히는 물리적 오류가 발생했습니다.
이를 해결하기 위해 발판 위에 생성되는 오브젝트들에 부모 발판 참조 변수를 주입하고, 발판에 닿거나 스폰되었을 때의 상대 좌표 오프셋을 계산하여 저장해 둔 뒤, 매 프레임 부모 발판의 왕복 운동 변위에 실시간으로 절대 좌표가 합산되어 추종하게 만드는 상대 좌표 동기화 물리를 설계하여 극복했습니다.

### **3. [3차 추가] 발판 끄트머리 판정 완화**
플레이어의 가로폭과 발판의 충돌 박스가 아주 미세하게 비껴갈 때, 플레이어가 허공에 발을 딛고 떨어져 즉사하는 부자연스러운 판정 오류가 있었습니다. 이를 해결하기 위해 플레이어 캐릭터 가로폭(거대화 크기 반영)에 25px의 소프트 마진 완화 영역(AABB Margin Softener)을 더해, 끝부분에 걸치더라도 안정적으로 착지 판정이 되도록 조작감을 부드럽게 보정했습니다.

## **10. 목표 변경 이력 및 사유 [3차 추가]**

1. **Fever Mode 트리거 변경 (콤보 방식 ──> 배터리 수집 방식)**
   * **이유**: 단순 콤보 숫자로만 무적이 켜지는 것보다, 맵 중간에 떠 있는 배터리 아이템을 5개 조화롭게 수집했을 때 켜지는 방식이 유저에게 더 높은 수집 목표와 보상을 부여할 수 있어 변경했습니다.
2. **대시 스택/충전 시간 제거 (스태미나 제한 ──> 무제한 대시 체이닝)**
   * **이유**: 대시 쿨타임을 기다리며 점프만 뛰는 구간에서 게임의 속도가 루즈해지는 문제를 발견했습니다. 대시 쿨타임을 과감히 없애 대시 연타를 통해 공중을 가르는 극적인 연쇄 대시 플레이가 가능하도록 재설계하여 타격감과 속도감을 높였습니다.
3. **새로운 특수 발판 기믹 추가 (지상 단순 질주 ──> 입체 지형 돌파)**
   * **이유**: 후반부로 갈수록 지형의 단조로움으로 인해 몰입도가 떨어져, 5주차에 좌우/상하 움직이는 발판, 무너지는 발판, 트램펄린 발판을 전면 추가 도입하여 플레이의 입체감을 불어넣었습니다.

## **11. 사용 기술 및 구체적인 개발 범위 [3차 추가]**

### **참고한 것들**
* **쿠키런 (Cookie Run)**: 2버튼 기반의 속도 지향적 모바일 조작 체계 및 피버 게이지 수집 시스템 참고.
* **소닉 더 헤지혹 (Sonic)**: 탄성 스프링 점프 물리 및 시간 차(0.8초) 붕괴형 플랫폼 추락 기믹 참고.
* **Android Developers API Document**: SoundPool 및 MediaPlayer의 비동기 재생 지연 최소화 및 가비지 컬렉터(GC) 리소스 관리 가이드 참고.

### **수업 내용에서 차용한 것**
* **Custom 2D Game Engine Architecture**: `GameContext`에 기반한 프레임 시간 보간 루프 설계 및 `SceneStack` 모델을 통한 씬 전환 제어.
* **Letterbox Resolution Scaling**: 기기 해상도 비율에 맞추어 레터박스를 자동 생성해 주는 `GameMetrics` 해상도 보정 기법.
* **Automated Lifecycle GC Management**: 화면 왼쪽 밖으로 이탈한 플랫폼, 적, 아이템 등의 객체를 실시간 모니터링하여 삭제 처리하는 파이프라인.

### **직접 개발한 것**
* **상대 좌표 연동형 움직이는 플랫폼 물리**: 왕복 운동을 수행하는 플랫폼의 요동치는 삼각함수 궤적에 맞춰 그 위에 얹어진 물체들이 실시간 위치를 추종하도록 설계한 물리 동기화.
* **베지어 궤적 추종형 대시 잔상 알고리즘**: 플레이어의 이전 좌표 리스트를 Queue 형태로 관리하며 대시 진행율에 비례한 굵기/불투명도로 리본 형태 잔상을 렌더링.
* **동적 자성 흡입 보간 물리**: 자성 임계 영역 진입 시 유클리드 거리에 기반한 역제곱 법칙 비율로 아이템 가속도를 조절해 캐릭터 방향으로 흡입시키는 견인 구현.
* **충돌 마진 보정 알고리즘**: 플레이어 캐릭터 가로폭에 소프트 마진 보정(25px)을 더해 불합리한 발판 낙사 억까 판정 보정.
* **동적 파티클 분산 시뮬레이터**: 적/가시 격파 시 흩어지는 사멸 연출용 2D 파티클 물리.
