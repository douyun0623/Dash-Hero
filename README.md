## **1. 게임 컨셉 (High Concept)**

* **High Concept**
  * **대시(Dash)** 와 **밟기(Stomp)** 를 통해 적을 밀어내며 전진하는 2D 러너 액션 게임입니다.
* **핵심 메카닉**
  * 화면 터치 시 전방으로 대시하여 적을 강하게 밀쳐냅니다.
  * 낙하 중 적의 머리를 밟으면 반동 점프로 다시 떠오릅니다.
  * 패턴 문자열 기반의 발판 생성으로 무한히 이어지는 지형을 구성합니다.
  * 대시 중에는 리본 형태의 **Dash Trail** 이펙트가 생겨 속도감을 보강합니다.

## **3. 실제 게임 실행 흐름 (Scene Flow)**

```text
앱 실행 (MainActivity)
  └─ MediaPlayer (BGM 재생) & SoundPool (SFX 준비)
  └─ TitleScene (시작 화면)
      ├─ 도시 배경 무한 스크롤 연출 & 반짝이는 "Tap to Start"
      ├─ HighScoreManager를 통한 로컬 최고 기록(Best Score) 출력
      └─ 화면 터치 시 ──> MainScene (인게임 플레이 시작)
          ├─ RUNNING (게임 진행 중)
          │   ├─ 우측 상단 '||' (Pause) 버튼 터치 ──> PauseScene (일시정지 화면)
          │   │                                       ├─ Resume ──> MainScene으로 게임 재개
          │   │                                       ├─ Restart ──> MainScene 신규 인스턴스로 즉시 재시작
          │   │                                       └─ Exit to Title ──> TitleScene 메인 화면으로 복귀
          │   └─ 적 충돌 / 낙사
          │       └─ GAME_OVER (결과 화면)
          │           ├─ 최신 점수 및 최고 점수 비교/업데이트
          │           └─ 화면 터치 시 ──> MainScene 즉시 재시작
```

## **4. MainActivity 구성**

```text
MainActivity (BaseGameActivity 상속)
 ├─ GameMetrics 가상 크기 설정: 900 x 1600 (Letterbox 자동 지원)
 ├─ SoundEffects (SoundPool): 점프, 대시, 스톰프, 아이템 획득, 게임오버 사운드 리소스 로드
 ├─ HighScoreManager (SharedPreferences): 최고 거리 점수 로컬 영구 로드
 ├─ MediaPlayer (BGM): 메인 배경음악(bgm_main.wav) 로드, 자동 루프 및 앱 활성 상태에 맞춰 재생/일시정지 제어
 └─ createRootScene()에서 TitleScene을 루트 화면으로 생성
```

## **5. MainScene의 Game Object 구성**

### **Player**
- **비주얼**: 둥근 노란색 사각형. 웅크림, 대시 등 상태에 따라 스쿼시/스트레칭 애니메이션 적용. 무적 복귀 시 투명하게 드로우.
- **물리**: 중력, 자동 점프(발판 Landing 시 Crouch 딜레이 후 탄성 상승), 대시 시 Y축 잠금 및 앞으로 뻗어 나가기, 복귀 감속 보간.
- **상호작용**: 발판 착지, 적 밟기 반동 상승, 대시 충돌 시 처치, 일반 상태 충돌 시 게임오버.

### **PlatformManager**
- **역할**: 지상 발판(`GroundPlatform`), 수집형 아이템(`Battery`), 지상 적(`Enemy`), 비행 드론 적(`DroneEnemy`)의 스폰 및 생명주기를 Procedural하게 관리.
- **스폰 규칙**: 7가지 무작위 패턴Preset(L, H, X, -) 기반으로 지형을 형성하며, 낙사구간 상공 및 플랫폼 위에 아이템과 적을 똑똑하게 분배 배치.

### **Battery (Item)**
- **비주얼**: Cyan 색상의 둥근 배터리 형상. 중앙에 노란색 번개 아이콘이 실시간으로 상하 부유 운동.
- **상호작용**: 플레이어와 충돌 시 수집 효과음(`playCollect`) 재생, 점수 보너스(+10m)를 지급하고 제거됨.

### **Enemy (지상 적)**
- **역할**: 발판 위에서 플레이어보다 낮은 높이로 통통 튀며 이동을 방해하는 보라색 적 캐릭터.
- **상호작용**: 머리를 밟히면 밟기 음 재생 및 플레이어 반동 상승. 대시 공격을 받으면 회전하며 우측 상단으로 날아가는 물리 사망 연출 적용.

### **DroneEnemy (비행 적)**
- **역할**: 삼각 비행 로봇 외형. 중앙에 노란 눈이 빛나며, 날개가 회전하는 모양. 공중 및 낙사 구간 위에서 사인파/코사인파 조합으로 부유 회전 정찰을 수행.
- **상호작용**: 대시 공격으로 제거하거나 머리 밟기가 가능하며, 정면 충돌 시 게임 오버 처리.

### **ParticleSystem**
- **시각효과**: 대시 공격 처치 시 주황/보라색 스파크 파티클 다수 스폰. 밟기 처치 시 노랑/보라색 파티클 스폰.
- **특징**: 스크롤 속도에 연동되어 왼쪽으로 같이 이동하며, 중력 없이 확산 및 투명도 페이드아웃 효과 적용.

## **6. 개발 이슈 및 어려웠던 점의 극복**

1. **지형 높낮이 차에 따른 클라이밍 불가 문제 극복**:
   - 무작위 발판 높낮이(L, H 오프셋)를 procedure하게 생성하는 과정에서 플레이어의 점프력 한계(최대 208px)보다 큰 고저차(최대 270px)가 생겨 진행 불가능한 병목이 발생했습니다.
   - 이를 해결하기 위해 플레이어 점프 초기 속도를 `jumpVelocity = -1200f`로 상향(점프 높이 300px 확보)하고, 플랫폼 오프셋 변위를 조율(최대 차이 220px)하여 모든 지형이 논리적으로 통과되도록 밸런스를 잡았습니다.
2. **투명 배경을 가진 일시정지 오버레이 구현**:
   - 일시정지 시 뒷배경(인게임)이 흐릿하게 깔리며 멈춰야 하는데, 프레임워크가 기본적으로 Top 씬 하나만 그리고 있었습니다.
   - `SceneStack`에 `getPreviousScene()` 참조 메서드를 구현하여 `PauseScene`이 호출될 때 렌더링 파이프라인 상에서 직전 인게임 씬을 먼저 그려주고 위에 반투명 어두운 사각형을 그리는 구조로 극적인 레이아웃 오버레이를 제작했습니다.
3. **복귀 페이즈 사망 루프 방지**:
   - 대시를 마치고 원래 위치로 복귀(Return)하는 도중 앞에 지나간 적의 옆면에 살짝 닿아 의문사하는 현상으로 조작감이 떨어졌습니다.
   - 복귀 페이즈 중에는 캐릭터의 투명도를 반투명(`alpha = 160`)하게 렌더링하고, 적들과의 충돌 검사 시 그냥 통과(Pass-through)하는 무적 프레임을 추가하여 시각적 개연성과 편안한 대시 연계를 지원했습니다.
